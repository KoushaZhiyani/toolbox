# SQL Insert Cleaner

A lightweight Python utility for cleaning and transforming SQL `INSERT INTO` statements.

This tool parses SQL dump files, removes unwanted columns based on configurable rules, optionally renames target tables, and generates a clean output SQL file while preserving the original values structure.

---

## Features

* Parse SQL `INSERT INTO` statements safely
* Remove columns using flexible exclusion rules
* Rename destination tables
* Preserve SQL functions and nested expressions
* Handle quoted strings correctly
* Process entire SQL dump files
* Support exact, prefix, suffix, and regex-based column filtering

---

## Supported Exclusion Rules

### Exact Match

Remove a specific column:

```python
"=APP_STATUS"
```

Removes:

```sql
APP_STATUS
```

---

### Suffix Match

Remove all columns ending with a suffix:

```python
"*_LABEL"
```

Removes:

```sql
STATUS_LABEL
CATEGORY_LABEL
PRODUCT_LABEL
```

---

### Prefix Match

Remove all columns starting with a prefix:

```python
"TEMP_*"
```

Removes:

```sql
TEMP_ID
TEMP_DATE
TEMP_VALUE
```

---

### Regular Expressions

Use Python regex patterns for advanced filtering:

```python
re.compile(r"^SYS_")
```

Removes:

```sql
SYS_ID
SYS_USER
SYS_DATE
```

## Usage

### Basic Example

```python
fix_insert_statements(
    "input.sql",
    "output.sql"
)
```

---

### Rename Target Table

```python
fix_insert_statements(
    "input.sql",
    "output.sql",
    target_table="new_table"
)
```

---

### Remove Multiple Columns

```python
fix_insert_statements(
    "input.sql",
    "output.sql",
    exclude_rules=[
        "*_LABEL",
        "=APP_STATUS",
        "=APP_NUMBER"
    ]
)
```

## Processing Pipeline

### 1. SQL File Splitting

The input SQL file is split into individual statements while respecting:

* Single quoted strings
* Double quoted strings
* Embedded semicolons inside text

---

### 2. INSERT Parsing

For every `INSERT INTO` statement:

* Extract table name
* Extract column list
* Extract VALUES section

---

### 3. Column Filtering

Columns are evaluated against exclusion rules.

Matching columns and their corresponding values are removed.

---

### 4. Statement Reconstruction

A new INSERT statement is generated using:

* Filtered columns
* Filtered values
* Optional target table name

---

### 5. Output Generation

All transformed statements are written to a new SQL file.
